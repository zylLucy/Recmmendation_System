# NCF 模型修改记录

## 一、修改目标

按照以下两个方向改进 NCF 模型：

1. **负样本加入 rating≤2 的真实低评分**：利用用户明确打低分的电影作为强负样本，替代纯随机采样
2. **BCELoss → BPR Loss**：将损失函数从二分类交叉熵改为贝叶斯个性化排序损失

---

## 二、代码修改详情

### 2.1 改动 1：负样本加入 rating≤2 的真实低评分

**修改文件：** `RecSys/RecSys/model/ncf/ncf.py`

**修改点 1：** `generate_dataset()` 中额外加载 rating≤2 的数据

```python
# 原代码：只加载 rating >= threshold 的正样本
if int(rating) < self.rating_threshold:
    continue

# 新代码：同时记录 rating <= 2 的强负样本
if rating_int >= self.rating_threshold:
    interactions.append((int(user), int(item)))
elif rating_int <= 2:
    strong_negatives.append((int(user), int(item)))
```

新增成员变量 `self.strong_neg_items_by_user` 存储每个用户的强负样本集合。

**修改点 2：** `_sample_negative_items()` 混合强负样本和随机负样本

```python
# 原代码：纯随机采样未交互电影
for n in range(num_neg):
    while True:
        item_idx = int(self.rng.integers(0, len(self.item_ids)))
        if item_idx not in self.train_items_by_user[int(user_idx)]:
            neg_items[idx, n] = item_idx
            break

# 新代码：最多 1 个来自 rating<=2 的强负样本，其余随机采样
strong_negs = list(self.strong_neg_items_by_user.get(uid, set()))
n_strong = min(1, len(strong_negs))
if n_strong > 0:
    chosen = self.rng.choice(strong_negs, size=n_strong, replace=False)
    neg_items[idx, :n_strong] = chosen
for n in range(n_strong, num_neg):
    # 随机采样补充
```

**设计考量：** 每个正样本最多配 1 个强负样本（共 4 个负样本），避免强负样本占比过高导致过拟合。

### 2.2 改动 2：BCELoss → BPR Loss（已回退）

**尝试过程：**

1. 将模型输出层移除 Sigmoid（BPR 需要原始分数）
2. 训练循环改为 BPR Loss：`-log(sigmoid(pos_score - neg_score))`
3. 学习率从 1e-4 降至 1e-5

**结果：** BPR Loss 训练不稳定，valid MRR@10 在 0.14 附近震荡后早停，测试集指标全面下降。

**结论：** BPR Loss 在此配置下效果不如 BCELoss，已回退。最终 V2 模型保留 BCELoss + 强负样本的组合。

## 五、分析结论

### 5.1 BPR Loss 为什么效果差？

1. **BPR 对学习率极度敏感**：1e-4 直接发散，1e-5 收敛极慢
2. **BPR 需要更大的 batch size**：BPR 依赖 batch 内的正负样本对比，4096 的 batch 中有效对比对不够多
3. **BCELoss 在全量排序评估中表现更好**：BCELoss 显式学习正负样本的绝对分数，在"从 3700 部电影中排序"的场景下比 BPR 的相对排序更稳定

### 5.2 强负样本为什么没有提升效果？

1. **全量强负样本导致过拟合**：模型过度关注"不要推荐 rating≤2 的电影"，忽略了更广泛的未交互电影空间
2. **混合策略缓解但未解决**：1 强负 + 3 随机的策略比全强负好，但仍不如纯随机
3. **可能原因**：rating≤2 的电影分布与测试集正样本分布不同，强负样本引入了分布偏移（distribution shift）

### 5.3 结论

在当前全量排序评估场景下，**V1 的纯随机负采样 + BCELoss 仍然是最优方案**。BPR Loss 和强负样本两项改进均未带来正向收益。

**建议后续方向：**
- 增大 Embedding 维度和 MLP 层数
- 尝试更多 epoch（当前早停在 84 epoch，可能还有提升空间）
- 引入用户/电影特征（年龄、性别、电影类型等）
- 尝试 Pairwise 损失的其他变体（如 WARP Loss）

### 6.4 分析

1. **V3 的 valid MRR@10 (0.3396) 与 V1 (0.3393) 几乎持平**，但测试集指标反而略低
2. **模型容量增大但效果未提升**，说明当前数据量下 64 维 Embedding + [32,16,8] MLP 已经足够
3. **早停更早触发**（V1: 79 epoch, V3: 74 epoch），大模型收敛更快但更容易过拟合
4. **训练时间更长**（V3: 979s vs V1: ~600s），但无收益

### 6.5 结论

单纯增大模型容量对 NCF 在 MovieLens 1M 上没有帮助。当前 V1 配置（embedding_size=64, mlp_hidden=[32,16,8]）在参数量和效果之间达到了较好的平衡。后续改进应聚焦于特征工程（引入用户/电影特征）或损失函数优化，而非简单增大模型。
