---

## 一、项目描述

本项目在 **MovieLens-1M** 数据集上实现并对比了三类推荐算法模型：

| 模型 | 类型 | 核心思想 |
|------|------|------|
| **DCN**（Deep & Cross Network） | 特征交叉网络 | 通过 Cross Network 显式学习高阶特征交叉，结合 Deep Network 学习隐式特征 |
| **NCF**（Neural Collaborative Filtering） | 神经网络协同过滤 | 用 GMF + MLP 双分支神经网络替代传统矩阵分解的内积，捕捉用户-物品的非线性交互 |
| **LightGCN** | 图神经网络 | 在图卷积网络中去除特征变换和非线性激活，简化 GCN 用于协同过滤 |

项目目标：通过统一的评估框架（8:1:1 数据划分、全量排序评估、早停机制），对比三类模型在推荐任务上的表现，并针对 NCF 模型进行多轮改进实验。

### 3.3 NCF（Neural Collaborative Filtering）

#### 3.3.1 模型原理

NCF（NeuMF）由 He 等人于 WWW 2017 提出，核心思想是用神经网络替代传统矩阵分解中的内积操作，捕捉用户-物品之间更复杂的非线性交互。

**模型结构：GMF + MLP 双分支融合**

```
UserID ─┬─ GMF Embedding ──┐
        │                  ├─ 逐元素乘积 ── [64维] ──┐
MovieID─┴─ GMF Embedding ──┘                        │
                                                    ├─ 拼接 ── FC ── Sigmoid ── 预测分数
UserID ─┬─ MLP Embedding ──┐                        │
        │                  ├─ 拼接 ── [64→32→16] ── [16维] ──┘
MovieID─┴─ MLP Embedding ──┘
```

- **GMF 分支**：逐元素乘积，捕捉线性交互（类似传统矩阵分解）
- **MLP 分支**：拼接后过三层全连接，捕捉非线性交互
- **融合层**：两个分支的输出拼接后过 FC + Sigmoid，输出 0~1 的喜欢概率

#### 3.3.2 核心代码解读

**（1）模型定义（NeuMF 类）**

```python
class NeuMF(nn.Module):
    def __init__(self, num_users, num_items, embedding_size=64,
                 mlp_hidden=(32, 16, 8), dropout=0.0):
        super().__init__()
        # GMF 分支的 Embedding
        self.gmf_user_emb = nn.Embedding(num_users, embedding_size)
        self.gmf_item_emb = nn.Embedding(num_items, embedding_size)
        # MLP 分支的 Embedding（独立参数，不与 GMF 共享）
        self.mlp_user_emb = nn.Embedding(num_users, embedding_size)
        self.mlp_item_emb = nn.Embedding(num_items, embedding_size)
        # MLP 层: 64*2 → 64 → 32 → 16
        mlp_modules = []
        input_dim = embedding_size * 2
        for out_dim in mlp_hidden:
            mlp_modules.append(nn.Linear(input_dim, out_dim))
            mlp_modules.append(nn.ReLU())
            if dropout > 0:
                mlp_modules.append(nn.Dropout(dropout))
            input_dim = out_dim
        self.mlp = nn.Sequential(*mlp_modules)
        # 融合层: GMF输出(64维) + MLP输出(16维) → 1
        fusion_dim = embedding_size + mlp_hidden[-1]
        self.predict = nn.Sequential(nn.Linear(fusion_dim, 1), nn.Sigmoid())

    def forward(self, user, item):
        # GMF: 逐元素乘积
        gmf_out = self.gmf_user_emb(user) * self.gmf_item_emb(item)
        # MLP: 拼接后过全连接
        mlp_out = self.mlp(torch.cat([self.mlp_user_emb(user),
                                       self.mlp_item_emb(item)], dim=-1))
        # 融合输出
        return self.predict(torch.cat([gmf_out, mlp_out], dim=-1)).squeeze()
```

**解读**：GMF 和 MLP 两个分支使用独立的 Embedding 层（不共享参数），这是论文中效果最好的配置。GMF 用逐元素乘积建模线性关系，MLP 用全连接层建模非线性关系，最后拼接融合。

**（2）训练循环（BCELoss + 负采样）**

```python
def train(self):
    criterion = nn.BCELoss()
    optimizer = optim.Adam(self.model.parameters(), lr=self.learning_rate)

    for epoch in range(1, self.epochs + 1):
        # 随机打乱训练数据
        order = self.rng.permutation(len(self.train_pairs))
        for start in range(0, len(order), self.batch_size):
            batch_idx = order[start:start + self.batch_size]
            users = self.train_pairs[batch_idx, 0]
            pos_items = self.train_pairs[batch_idx, 1]

            # 每个正样本采样 num_neg 个负样本
            neg_items = self._sample_negative_items(users, self.num_neg)

            # 构建 batch: 正样本(label=1) + 负样本(label=0)
            batch_users = np.concatenate([users] * (1 + self.num_neg))
            batch_items = np.concatenate([pos_items] +
                [neg_items[:, i] for i in range(self.num_neg)])
            batch_labels = np.concatenate(
                [np.ones(len(users)), np.zeros(len(users) * self.num_neg)])

            # 前向 + 反向传播
            preds = self.model(users_t, items_t)
            loss = criterion(preds, labels_t)
            loss.backward()
            optimizer.step()
```

**解读**：采用隐式反馈范式，将推荐问题转化为二分类问题。每个正样本配 4 个随机负样本，正负比 1:4。使用 BCELoss 训练模型区分"喜欢"和"未交互"。

#### 3.3.4 模型改进历程

在 NCF 模型上进行了三轮改进实验：

**V1（原始）**：纯随机负采样 + BCELoss，作为基线模型。

**V2（强负样本 + BPR Loss）**：尝试了以下方案：

| 方案 | 损失函数 | 负样本策略 | Valid MRR@10 | 结论 |
|------|------|------|------|------|
| V1 基线 | BCELoss | 纯随机 | 0.3393 | 基线 |
| BPR-1e-4 | BPR | 纯随机 | 0.1371 | 不收敛 |
| BPR-1e-5 | BPR | 纯随机 | 0.1440 | 收敛慢，效果差 |
| BCELoss-全强负 | BCELoss | 4 个全用 rating≤2 | 0.1349 | 过拟合 |
| BCELoss-混合 | BCELoss | 1 强负 + 3 随机 | 0.2881 | 不如 V1 |

**结论**：BPR Loss 对学习率极度敏感且收敛慢；强负样本引入分布偏移导致过拟合。两项改进均未带来正向收益，已回退。

**V3（增大模型容量）**：将 embedding_size 从 64 增至 128，mlp_hidden 从 [32,16,8] 增至 [64,32,16]，模型参数量约 4 倍。

**结论**：所有指标反而略降（MRR@10: 0.3393 → 0.3359），训练时间更长（979s vs ~600s）。说明当前数据量下 64 维 Embedding 已经足够，单纯增大模型无益。

### 4.5 指标分析

**NCF 模型表现解读：**

1. **HIT@10 = 0.6767**：67.7% 的用户在 Top-10 推荐中至少命中 1 个喜欢的电影。考虑到评估是从 3,628 部电影中全量排序（而非采样），这个命中率是合理的。

2. **MRR@10 = 0.3393**：第一个喜欢的电影平均排在第 3 位（1/0.3393 ≈ 2.95），说明模型排序质量较好，喜欢的电影倾向于排在前面。

3. **RECALL@10 = 0.1535**：平均召回了 15.35% 的测试集正样本。Recall 偏低是因为每个用户的测试集正样本较多（平均约 16 个），而只取 Top-10，分母较大。

4. **NDCG@10 = 0.1892**：考虑了命中位置的加权指标，偏低说明命中位置还可以进一步优化。

5. **PRECISION@10 = 0.1426**：Top-10 推荐中约 14.3% 是用户真正喜欢的，在全量排序场景下属于正常水平。

**改进实验的启示：**

- BPR Loss 在全量排序评估下不如 BCELoss，因为 BCELoss 显式学习绝对分数，更适合"从大量候选中挑出正样本"的场景
- 强负样本（rating≤2）引入分布偏移，模型过度关注"不要推荐低分电影"而忽略更广泛的未交互空间
- 增大模型容量（128 维 Embedding）在 MovieLens-1M 上无收益，当前 64 维已足够

1. **V1 在所有指标上均为最优**：纯随机负采样 + BCELoss 的组合在 MovieLens-1M 全量排序评估场景下表现最好。

3. **V3 指标略降**：增大模型容量（embedding_size 64→128, MLP [32,16,8]→[64,32,16]）反而导致轻微过拟合，说明当前数据量下 64 维 Embedding 已足够。

4. **HIT@10 始终在 0.59 以上**：说明 NCF 模型具有较强的用户偏好捕捉能力，即使改进方案效果下降，仍有近 60% 的用户能在 Top-10 中命中喜欢的电影。

### 5.5 总结、感想与收获

#### 总结

本项目在 MovieLens-1M 数据集上实现了 NCF 推荐模型，并进行了三轮改进实验。主要发现：

1. **NCF 模型在 MovieLens-1M 上表现稳定**：V1 配置下 HIT@10=0.68, MRR@10=0.34，说明模型能有效学习用户偏好。

2. **负采样策略对模型影响显著**：纯随机负采样反而优于引入真实低评分作为强负样本，因为强负样本引入了分布偏移。

3. **损失函数选择需匹配评估场景**：BCELoss 在全量排序评估下优于 BPR Loss，因为 BCELoss 显式学习绝对分数。

4. **模型容量并非越大越好**：在 MovieLens-1M 数据量下，64 维 Embedding + [32,16,8] MLP 已达到较好平衡。

#### 感想与收获

1. **推荐系统评估的复杂性**：离线评估（全量排序）与在线评估（用户真实反馈）之间存在差距，离线指标好不一定代表线上效果好。

2. **负采样是推荐系统的核心问题**：如何选择高质量的负样本直接影响模型效果，简单的随机采样有时反而是最优解。

3. **改进不一定带来提升**：BPR Loss 和强负样本在理论上更优，但在实际场景中效果反而不如简单的 BCELoss + 随机负采样。这提醒我们在做模型改进时要保持实验验证的习惯。

4. **数据质量决定上限**：MovieLens-1M 数据质量高（无缺失、无异常），使得我们可以专注于模型本身的优化，而无需花费大量精力在数据清洗上。

5. **统一评估框架的重要性**：项目通过统一的 8:1:1 划分、全量排序评估、早停机制，确保了不同模型和不同版本之间的公平对比。
