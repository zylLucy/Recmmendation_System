# MovieLens-1M 对齐参数表

本表用于让 XSimGCL、LightGCN 以及其它协作实现的同学使用同一套数据、切分、训练和评估口径。参考来源是 RecBole-GNN `results/general/ml-1m.md`，本地 XSimGCL 已按这个协议做了对齐。

## 统一实验口径

| 项目 | 对齐值 | 说明 |
| --- | --- | --- |
| 数据集 | MovieLens-1M | 使用 `data/ml-1m/ml-1m/ratings.dat`、`users.dat` |
| 交互过滤 | `rating >= 3` | 官方写法是 remove rating score less than 3 |
| 用户节点 | 6040 | 从 `users.dat` 保留全量用户；用户 3598 无正反馈，作为孤立用户保留 |
| 物品节点 | 本地正反馈物品 3628 | 官方 RecBole 统计表写 3629，通常含内部占位/映射口径；本地图节点按过滤后正反馈物品建图 |
| 正反馈交互 | 836478 | 过滤后交互总数 |
| 切分方式 | per-user ratio split | 每个用户内部随机切分，不做全局 shuffle 后切分 |
| 切分比例 | `8:1:1` | train/valid/test |
| 排序评估 | full sort | 对所有候选物品排序，mask 已知训练物品；test 时也 mask valid |
| TopK | `10` | 统一报告 `@10` |
| 验证指标 | `MRR@10` | 按 valid MRR 选择 best checkpoint |
| 测试 checkpoint | best valid checkpoint | 不直接用最后一轮 |
| 指标口径 | mean per user | `Recall/MRR/NDCG/Hit/Precision@10` 均按用户计算再平均 |
| 随机种子 | `2020` | 本地默认值；其他实现也建议固定 |
| epoch | `500` | 快速调试可降低，正式对齐用 500 |
| train batch size | `4096` | BPR/图协同过滤类统一 |
| embedding size | `64` | 官方公平比较设置 |
| eval batch size | full-sort capacity | RecBole 配置为 `4096000`，本地实现可直接全量打分 |

本地 XSimGCL 当前数据构建输出应类似：

```text
用户数:6040，电影数:3628，交互数:836478，训练/验证/测试:675255/80902/80321
```

因为是 per-user split，各用户内部取整后，全局 train/valid/test 数量不会刚好等于 `80%/10%/10%`。

## 模型超参数

| 模型 | 官方 best hyper-parameters | 对齐备注 |
| --- | --- | --- |
| BPR | `learning_rate=0.001` | 统一 `embedding_size=64`、`epochs=500`、`train_batch_size=4096` |
| NeuMF | `learning_rate=0.0001`<br>`mlp_hidden_size=[32,16,8]`<br>`dropout_prob=0` | 保持同一数据切分；若实现含 GMF/MLP embedding，也使用 64 维主 embedding |
| NGCF | `learning_rate=0.0002`<br>`message_dropout=0.0`<br>`node_dropout=0.0` | 统一 `embedding_size=64` |
| LightGCN | `learning_rate=0.002`<br>`n_layers=3`<br>`reg_weight=0.0001` | LightGCN 同学重点按这三个参数对齐 |
| LightGCL | `learning_rate=0.001`<br>`n_layers=2`<br>`lambda1=0.0001`<br>`temp=2`<br>`lambda2=1e-7`<br>`dropout=0.1` | 参数名可能随实现不同而变化，保持语义一致 |
| SGL | `learning_rate=0.002`<br>`n_layers=3`<br>`reg_weight=0.0001`<br>`ssl_tau=0.5`<br>`drop_ratio=0.1`<br>`ssl_weight=0.005` | SSL 温度和边/节点 dropout 要和实现中的增强方式对应 |
| HMLET | `learning_rate=0.002`<br>`n_layers=4`<br>`activation_function=leakyrelu` | 保持图层数和激活函数一致 |
| NCL | `learning_rate=0.002`<br>`n_layers=3`<br>`reg_weight=0.0001`<br>`ssl_temp=0.1`<br>`ssl_reg=1e-06`<br>`hyper_layers=1`<br>`alpha=1.5` | 注意 `ssl_reg` 和普通 `reg_weight` 不是同一个参数 |
| SimGCL | `learning_rate=0.002`<br>`n_layers=2`<br>`reg_weight=0.0001`<br>`temperature=0.05`<br>`lambda=1e-5`<br>`eps=0.1` | `lambda` 是对比学习损失权重 |
| XSimGCL | `learning_rate=0.002`<br>`n_layers=2`<br>`reg_weight=0.0001`<br>`temperature=0.2`<br>`lambda=0.1`<br>`eps=0.2`<br>`layer_cl=1` | 本地 `ssl_weight` 对应官方 `lambda` |

## 实现检查清单

| 检查项 | 要求 |
| --- | --- |
| 数据过滤 | 只保留 `rating >= 3` |
| 用户保留 | 从 `users.dat` 建 6040 个用户节点，不因无正反馈删除用户 |
| 切分 | 每个用户内部随机切 `8:1:1` |
| 训练图 | 只使用 train 交互建图 |
| valid 评估 | mask train，评估 valid，按 `MRR@10` 保存 best |
| test 评估 | 加载 best checkpoint，mask train + valid，评估 test |
| 负采样 | 训练负样本不能在该用户 train 正样本中 |
| full sort | 不使用 sampled evaluation 报最终分数 |
| 指标 | 按用户平均，不用全局 hit / 全局 test item 作为 Recall |
| 输出 | 推荐保存 `Recall/MRR/NDCG/Hit/Precision@10`，可额外保存 MAP |
